# ailc ROADMAP и честный статус зрелости

Этот файл честно фиксирует, что в ailc сделано и подтверждено, что сделано узко или
скелетом, и что ещё впереди. Источник истины это код и тесты; файл служит картой.

## Сделано и подтверждено (код и тесты)

- Детерминированный вердикт качества: гарантию даёт гейт, а не нейросеть. Минимум гейта
  по безопасности неотключаем, политика проекта сверяется с эталоном и помечает ослабление.
- Глубокий межпроцедурный анализ заражения (taint) доказан матрицей по пятнадцати языкам
  для трёх классов инъекций (исполнение команды, SQL, путь к файлу), с учётом санитайзеров.
- Статический разбор по абстрактному синтаксическому дереву (AST-SAST), правила OWASP и
  Web/API, каждое сообщение со ссылкой на CWE и OWASP.
- Чистка ложных срабатываний без потери реального сигнала: дедуп задвоенных правил,
  точность по контексту HTTP-клиента, правила доступности только по элементам разметки,
  исключение тест-кода и точек входа фреймворков, исключение защищённых XML-парсеров.
- Вывод SARIF 2.1.0 для непрерывной интеграции: в отчёт идут только подтверждённые находки,
  ложные опровергнуты, число опровергнутых и пропуски видны в свойствах.
- Definition of Done: многоосевой детерминированный вердикт «готово ли к сдаче».
- Правила сигналов соответствия требованиям РФ (152-ФЗ, 242-ФЗ, 187-ФЗ и ГОСТ) как сигналы
  для ревью.
- Лицензия Apache 2.0 на открытое ядро, оговорка о границах соответствия.
- Непрерывная интеграция: сборка release и тесты на каждый push и pull request.

## Сделано узко или скелетом (работает, но меньше полного замысла)

- Генерация документации: заготовки по ГОСТ 19.201 и 34.602 и по arc42 это скаффолд с
  авто-разделами из кода, а не полный документ; C4 это один граф Mermaid, а не полные
  уровни «Контекст, Контейнер, Компонент».
- Движки генерации, метрик, диаграмм, индекса и запуска это каркасы, а не полная реализация.

## Впереди (дорожная карта открытого ядра)

- Полная генерация документов по стандартам вместо скаффолда и полные уровни C4.
- Углубление каркасных движков.
- Расширение покрытия языков и правил.

## Коммерческие дополнения (по отдельной лицензии, вне открытого ядра)

Открытое ядро находит и показывает сигналы. Платные дополнения распространяются отдельно
по коммерческой лицензии и в этот репозиторий не входят: аттестационный отчёт по комплаенсу
(превращает сигналы в документ для аудитора со статусами требований и доказательствами),
уровень команды (центральная панель, сведение находок по многим репозиториям, блокировка
запросов на слияние в конвейере) и уровень предприятия (локальный управляющий слой,
разграничение доступа, журнал аудита, поддерживаемые пакеты соответствия как подписка).

## Известные ограничения

- Анализ заражения теряет поток через поля объектов и коллекции; ветвления не раздваивают
  состояние (консервативно); санитайзеры из курируемого списка контекст-независимы.
- Часть правил Web построчны; модель памяти C и C++ упрощена.
- Сервер протокола MCP при работе в среде разработки может держать кэш состояния, поэтому
  после правки файлов вердикт через MCP стоит сверять со свежим прогоном из терминала
  (`ailc dod .`), пока кэш не сброшен перезапуском.
- Несколько стилевых предупреждений clippy (не ошибки), не влияют на сборку и тесты.
