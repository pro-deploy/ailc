# ailc (AI Life Cycle)

**Автономный оркестратор качества для не-экспертов** (вайбкодер и энтерпрайз-джун).
Человек задаёт **намерение простым языком** — сервер сам понимает код, подбирает и
гонит проверки, чинит безопасное, ведёт документацию и выносит вердикт качества,
эскалируя человеку только настоящие решения. **Гарантию даёт гейт, а не нейросеть.**
Rust, один защищённый бинарь со встроенной мультиязычной моделью, без внешних сервисов.

Поток: `намерение/наблюдение → планировщик → пайплайн (DAG, параллельно) → verify-проход → гейт (governance) → вердикт человеку`.

## Возможности
- **9 движков**, ~59 инструментов (capability = тонкий конфиг поверх движка, без дублирования).
- **3-тирный роутер**: LLM клиента (MCP sampling) → семантика (встроенные эмбеддинги, офлайн) → ключевые слова. Каждый с откатом.
- **verify-максимализм**: состязательный проход опровергает ложные находки (секрет в комментарии/плейсхолдер), **Rigor Score** = тщательность; в балл идёт только верифицированное.
- **Governance как данные**: `ailc.policy.toml` авторит старший — джун наследует автоматически.
- **Покрытие**: web/backend + **mobile** (Flutter/Android/iOS/Swift) + **desktop** (.NET/Tauri/Electron/C++), полиглот (Go/Rust/Python/TS/JS/Java/Kotlin/Swift/Dart/C#).
- **Честность**: нет молчаливых пропусков (всё пропущенное — с причиной: скрытое/служебное/блобы считаются и видны в сводке), сбой инструмента ≠ находка, Rigor=0 при нуле выполненных проверок.
- **OSV офлайн**: Cargo.lock · package-lock.json · requirements.txt · go.sum · gradle.lockfile · pubspec.lock · Podfile.lock; экосистемы без записей в базе помечаются явно («не покрыто» ≠ «чисто»).

### Безопасность (глубокий слой)
- **SAST + taint-анализ**: структурный AST-SAST (инъекции/SQL/десериализация) и **межпроцедурный taint** на **всех 15 языках движка** (Python/JS/TS/Go/Java/Ruby/PHP/C#/Rust/Kotlin/Scala/C/C++/Swift/Dart) — недоверенный ввод → опасный сток (исполнение команды · SQL · открытие файла · переполнение буфера для C/C++) через цепочку присваиваний и **границы функций**, с учётом **санитайзеров** (`escape`/`shlex.quote`/`escapeshellarg`/… снимают заражение). Для C/C++ — своя модель угроз (`getenv`/`argv`/`fgets`→`strcpy`/`sprintf`/`system`, CWE-120/134). По классу — уровень CodeQL/Semgrep Pro, целиком офлайн.
- **Web/API правила**: SSRF, SSTI, XXE, open-redirect, path-traversal, небезопасная десериализация, отключённый TLS-verify, CORS-wildcard, JWT-none, GraphQL-интроспекция, mass-assignment — каждое сообщение с проверенной ссылкой **CWE + OWASP**.
- **AI/LLM-безопасность** (`security.ai/*`): промпт-инъекция и небезопасная обработка вывода модели (**OWASP LLM01/LLM02**); ключи LLM-провайдеров (OpenAI/Anthropic/HuggingFace).
- **Комплаенс РФ**: 152-ФЗ (ПДн в логах, предзаполненное согласие), 242-ФЗ (локализация), трансгранична (иностранные трекеры), **ГОСТ-крипта на объектах КИИ** (187-ФЗ).
- **SARIF-вывод** (`ailc sarif`): отчёт 2.1.0 для CI / GitHub security-tab — в отчёт идут **только верифицированные** находки (ложные отсеяны Verifier'ом), а число опровергнутых и пропуски видны в `properties`.
- **Экспорт навыков** (`ailc skills`): все возможности как **agentskills.io-пак** (SKILL.md + plugin.json) — discoverable для Claude Code / Cursor; навыки вызывают реальный движок через MCP.

### Документация, спека, архитектура — живой слой (из кода)
- **Страж недоделанного**: ловит заглушки (`unimplemented!`/`TODO()`/`NotImplementedError`), пустые обработчики/функции и недокументированное публичное API (`quality.check/completeness`, `quality.check/undocumented`). Незавершённое **блокирует на сдаче** (релиз/`ailc dod`), в мид-билде — предупреждение.
- **Генерация из кода** (`generate/spec|architecture|c4|data-model|glossary`): спецификация по **ГОСТ 19.201/34.602**, архитектура по **arc42**, диаграммы **C4** (Context/Container/Component, Mermaid), модель данных, глоссарий. Идемпотентно: код-разделы в авто-блоке, человеческие — скаффолд (ручные правки переживают регенерацию).
- **Извлечение поверхности** (`code.intel/surface`): HTTP-эндпоинты (FastAPI/Flask/Express/Gin/Spring/axum/Django/Rails/NestJS/ASP.NET/Laravel), ENV-переменные, внешние сервисы (БД/очереди/S3), модели данных (Prisma/SQL/ORM).
- **Дрейф доков в гейте** (`spec.check/drift`): сверяет авто-блок документа со свежими фактами из кода → «доки устарели/отсутствуют» рядом с безопасностью; видно в вердикте как «📋 Советы». Custodian держит доки в синхроне сам и проектирует недостающее.
- **Проектирование фичи** (`ailc design "<фича>"`): заготовка спеки (критерии приёмки · затрагиваемые части из карты кода) + **ADR** (Nygard).

- 622 теста (taint по 15 языкам · страж недоделанного · генерация и дрейф доков · регрессионный корпус ложных срабатываний) · непрерывная интеграция (сборка и тесты) в GitHub Actions.

## Сборка
```sh
cargo build --release      # → target/release/ailc (один бинарь ~72 МБ, strip+LTO, модель + 15 AST-грамматик внутри)
```

## Режимы
```sh
ailc <путь> "хочу выкатить фичу"   # разовый прогон → QualityLedger (вердикт человеку)
ailc serve                         # MCP-сервер поверх stdio (для IDE)
ailc custodian <путь> [N] [--fix]  # непрерывно: наблюдай→[почини]→проверь→доки→эскалируй
ailc dod <путь>                    # Definition of Done — многоосевой вердикт ✓/✗
ailc design "<фича>" <путь>        # спроектировать новую фичу: заготовка спеки + ADR
ailc fix <путь>                    # безопасная починка формата/линта (свои авто-фиксеры)
ailc sarif <путь> > out.sarif      # SARIF 2.1.0 для CI (только верифицированные находки)
ailc skills [папка]                # экспорт возможностей как agentskills-пак
ailc cap <id> <путь> [query]       # dev: прогнать один инструмент
```

## Подключение в IDE (MCP)
`.mcp.json` (Claude Code), `~/.cursor/mcp.json` или аналог:
```json
{
  "mcpServers": {
    "ailc": { "command": "/путь/к/target/release/ailc", "args": ["serve"] }
  }
}
```
Наружу — **4 front-door инструмента** (каталог из ~59 capability скрыт за роутером):
- `plan {intent, path}` — автономно: намерение → пайплайн → вердикт.
- `find_capability {query}` — семантический подбор инструмента под задачу (+ схема входа).
- `run {id, path, target?, query?}` — выполнить конкретный инструмент.
- `autofix {path, max?}` — LLM-починка с адверсариальной перепроверкой (нужен sampling).

Ответы содержат `structuredContent` (findings/ledger как объекты) рядом с человеко-текстом.
Если клиент поддерживает sampling — планировщик использует его LLM (иначе семантика/ключевые слова).

## Governance (правила компании)
Старший кладёт в корень проекта `ailc.policy.toml`:
```toml
name = "my-org"
[gate]
block_at = "high"                  # info | low | medium | high | critical
families = ["security", "quality"]
```
Джун ничего не настраивает — гейт применяет правила сам.

## Архитектура
Инструмент = `{family, engine, config}`. Движки реализованы один раз, capability — конфиги над ними.

| Слой | Крейт |
|---|---|
| Контракты (типы) | `ailc-contracts` |
| Движки · оркестратор · planner/router/verify · governance | `ailc-core` |
| Capability (конфиги поверх движков) | `ailc-capabilities` |
| Бинарь · MCP-транспорт · custodian/dod/fix | `ailc` |

Движки (`ailc-core/src/engines/`): Scan · CodeIntel · Gate · Runner · Generator · Store · Metric · Diagram · Index (эмбеддинги) · Surface (извлечение эндпоинтов/ENV/сервисов/моделей из кода).

## Честные ограничения
- Автофикс — только формат/линт (свои фиксеры проекта); security/логику решает человек.
- CodeIntel — AST основной слой (tree-sitter, 15 языков, вкл. Kotlin/Swift/Dart); regex — фолбэк при ошибке парсера.
- Шаг пайплайна ограничен 180с: зависшая проверка получает явный «превысил лимит», а не блокирует прогон.
- `target` из MCP-входа валидируется: `..`/абсолютные пути не выводят сканирование за корень проекта.
- При самосканировании security-тулом своих файлов с паттернами возможны само-срабатывания (как у любого сканера) — на реальных проектах их нет.
- Taint-анализ: межпроцедурный по source-функциям (хелпер, возвращающий ввод), внутри функции — поток-чувствительный; ветвления не раздваивают состояние (консервативно). Санитайзеры из курируемого списка контекст-независимы (shell-quote считается чистым и для SQL) — компромисс в пользу меньшего числа FP.

Состояние проекта, что сделано и что осталось — в `ROADMAP.md`.

Корпоративные и комплаенс-модули (поддерживаемые пакеты соответствия, управляющий слой предприятия, аттестация) распространяются по отдельной коммерческой лицензии.

## Границы соответствия
Проверки соответствия требованиям российского законодательства (152-ФЗ, 242-ФЗ, 187-ФЗ/ГОСТ) это **сигналы для ревью, а не юридическая гарантия**. Инструмент помогает обнаруживать и подтверждать признаки несоответствия в коде, но не заменяет ни юридическую ответственность организации, ни организационные меры (размещение данных, уведомление регулятора, категорирование объектов критической информационной инфраструктуры), ни аттестацию. Ответственность за соблюдение закона остаётся на организации.
